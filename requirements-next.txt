# WHY THIS FILE EXISTS

# The pinned requirements.txt (transformers 4.51.3 / vLLM 0.8.5) is a known-good
# set that REPRODUCES every number already committed to the deck (OLMo-2,
# Qwen2.5, Qwen3, Gemma-2, the Track-2 sweep). It CANNOT load any 2026-generation
# model: Qwen3.5/3.6, Gemma-4, Llama-4, OLMo-Hybrid, Granite-4, Nemotron-3 all
# use model_type / multimodal / hybrid-SSM architectures unknown to those libs.
#
# Rather than bump the pinned set in place (which would put the committed numbers
# at reproducibility risk), the next-gen ladders run in a SEPARATE environment
# built from this file. Two-env split:
#   * .venv        (requirements.txt)       -> reproduces the committed results
#   * .venv-next   (requirements-next.txt)  -> runs NEXTGEN_LADDER_KEYS only
# Select next-gen ladders with ATAX_LADDERS=qwen35_9b,qwen36_27b,... and point
# the run at the .venv-next interpreter.
